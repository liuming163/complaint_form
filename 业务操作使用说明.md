# 业务操作使用说明

## 一、系统用途

本系统用于投诉业务的资料管理与自动化提交，当前首页主要包含：
- 投诉账号管理
- 被代理人信息管理
- 作品覆盖列表
- UC 投诉
- 首页状态查询

建议每次正式投诉前，先按顺序完成以下检查：
1. 检查投诉账号是否已配置、Cookie 是否有效
2. 检查被代理人信息是否已覆盖且资料完整
3. 检查当前投诉作品是否已覆盖
4. 再进入 UC 投诉页面发起投诉

---

## 二、首页功能说明

进入系统首页后，可看到两个主要区域：

### 1. 顶部导航区
顶部可直接进入以下页面：
- **投诉账号管理**：维护投诉平台、投诉账号、Cookie、账号用途
- **被代理人信息管理**：维护被代理人名称、营业执照、授权委托书等资料
- **作品覆盖列表**：维护作品名称、代理主体(司内)、被代理人信息、内容类型、投诉类型、作品权属文件等

### 2. 左侧「投诉平台」区域
用于进入实际投诉页面：
- **夸克投诉**：夸克平台投诉表单
- **UC投诉**：UC 平台投诉表单

### 3. 右侧「状态查询」区域
用于查看任务执行状态：
- **夸克状态查询**：当前暂无数据展示
- **UC状态查询**：可查看 UC 投诉任务的提交时间、采集账号、作品名称、投诉单号、脚本执行状态、预估完成时间、日志入口

#### UC 状态查询说明
- 点击 **刷新** 可拉取最新状态
- 当任务处于 **等待中 / 执行中 / 已完成 / 失败** 时，会显示不同状态
- 若日志仍可用，可点击 **查看日志** 排查问题
- 如果显示 **日志已过期**，说明日志保留期已过

---

## 三、投诉账号管理

### 1. 页面用途
用于维护实际发起投诉所使用的平台账号信息，包括：
- 平台名称
- 代理主体(司内)
- 投诉账号
- 账号用途
- Cookie

### 2. 页面字段说明
账号列表中展示：
- **平台名称**：当前支持 UC、夸克
- **代理主体(司内)**：如和晞科技、柏蒙文化
- **投诉账号**：实际登录投诉平台的账号
- **账号用途**：便于区分账号使用场景
- **Cookie**：投诉时自动带入，用于身份验证。列表中只展示开头与结尾的若干位、中间用省略号；鼠标悬停可看完整内容

### 3. 操作说明
#### （1）新增投诉账号
如果未添加投诉平台和投诉账号，需要先在本页面添加：
1. 点击 **添加账号**
2. 选择 **代理主体(司内)**
3. 选择 **平台名称**（UC / 夸克）
4. 填写 **投诉账号**
5. 粘贴完整 **Cookie**
6. 选填 **账号用途**
7. 点击 **确认添加**

#### （2）更新 Cookie
如果账号已存在，但 Cookie 失效或需要更换：
1. 在列表中找到对应账号
2. 点击 **更新Cookie**
3. 粘贴新的完整 Cookie
4. 点击 **确认更新**

#### （3）展示规则
- 表格内容**不换行**展示，过宽时使用页面**横向滚动条**
- Cookie 列默认缩略，方便整体浏览，鼠标悬停可看完整 Cookie

### 4. 使用要求
进行 UC 投诉前，**必须先检查本页面**：
- 是否已经添加了需要使用的平台和投诉账号
- 当前账号对应的 Cookie 是否仍然有效

如果没有配置账号或 Cookie 已失效，UC 投诉无法正常提交。

## 四、被代理人信息管理

### 1. 页面用途
用于维护投诉时所使用的被代理人（权利人）信息，以及相关证明资料。

### 2. 页面展示内容
页面列表按账号维度展示：
- 平台名称
- 代理主体(司内)
- 投诉账号
- 账号用途
- 被代理人信息
- **被代理人营业执照**：列表中显示文件名，点击可在新窗口预览
- **授权委托书**：列表中显示文件名，点击可在新窗口预览

表格内容不换行展示，超宽时支持横向滚动。

### 3. 添加被代理人流程
进行 UC 投诉前，需要检查当前要使用的被代理人信息是否已覆盖。

如果未覆盖：
1. 点击 **添加被代理人**
2. 选择 **代理主体(司内)**
3. 选择 **平台名称**
4. 选择 **投诉账号**
5. 输入 **被代理人信息**
6. 按需上传资料：
   - 被代理人营业执照
   - 授权委托书
   - 授权期限截止日期
7. 点击 **确认添加**

#### 上传文件命名规则（重要）
为了避免错传，上传时会**对文件原始名称做即时校验**：

- **被代理人营业执照** 必须命名为：
  ```
  营业执照_被代理人信息.xxx
  ```
  例如：`营业执照_深圳阅荣信息科技有限公司.png`

- **授权委托书** 必须命名为：
  ```
  授权委托书_被代理人信息_代理主体(司内)_截止日期YYYYMMDD.xxx
  ```
  例如：`授权委托书_深圳阅荣信息科技有限公司_和晞科技_截止日期20260605.pdf`

如果文件名不符合规则：
- 选择文件后会**立即在表单内提示错误**
- 不允许提交
- 提交时后端也会再次校验

> 系统在落盘时仍会按内部统一规则重命名保存，不影响原有目录结构。

### 4. 页面逻辑说明
- 被代理人资料是按 **平台 + 代理主体(司内) + 投诉账号 + 被代理人名称** 进行管理的
- 如果已有部分资料，页面会提示已存在资料
- 已存在的资料在本页面中只展示，不允许直接覆盖上传
- 缺失部分可以继续补充

### 5. 投诉前检查要求
进行 UC 投诉前，必须先检查：
- 当前要使用的 **被代理人信息是否已覆盖**
- 被代理人对应的 **营业执照 / 授权委托书 / 授权期限** 是否正确
- 被代理人是否与本次投诉模版、作品资料保持一致

如果没有覆盖，需先新增后再投诉。

## 五、作品覆盖列表

### 1. 页面用途
用于维护当前可用于投诉的作品资料。

每条作品资料至少关联：
- 剧名
- 代理主体(司内)
- 被代理人信息
- 内容类型
- 投诉类型
- 作品权属文件
- 其他证明文件

### 2. 列表说明
页面列表可查看：
- **剧名**
- **代理主体(司内)**
- **被代理人信息**
- **内容类型**
- **投诉类型**
- **作品权属文件**：文件名可点击，新窗口预览
- **其他证明**：当存在其他证明文件时，会显示 `查看(N)` 按钮，点击后弹出列表，每个文件都可点击预览

支持搜索：
- 剧名
- 代理主体(司内)
- 被代理人信息
- 内容类型
- 投诉类型

> 同一作品允许同时存在多个被代理人，记录会按 **作品名称 + 代理主体(司内) + 被代理人信息 + 内容类型 + 投诉类型** 区分。

### 3. 新增作品流程
进行 UC 投诉前，需要检查当前需要投诉的作品是否已覆盖。

如果未覆盖：
1. 点击 **新增作品**
2. 填写 **剧名**
3. 选择 **代理主体(司内)**
4. 选择 **被代理人信息**（下拉选项来自当前代理主体下已建立的被代理人资料）
5. 选择 **内容类型**
6. 选择 **投诉类型**
7. 上传 **作品权属文件**（必传）
8. 按需上传 **其他证明文件**（最多 2 个，非必填）
9. 点击 **提交**

弹窗中除“是否还有其他证明文件”外，**其他选项均为必填项**。

#### 上传文件命名规则（重要）
为了避免错传，弹窗中会**对上传文件原始名称做即时校验**：

- **作品权属文件** 必须以下列前缀开头：
  ```
  证明文件_作品名称
  ```
  例如均允许：
  - `证明文件_铳火.png`
  - `证明文件_铳火_file.jpg`
  - `证明文件_铳火_-.jpg`

- **其他证明文件** 必须严格符合：
  ```
  其他证明_作品名称_序号.xxx
  ```
  序号必须是阿拉伯数字。例如：
  - `其他证明_铳火_1.png`
  - `其他证明_铳火_2.jpg`

  不允许：
  - `其他证明_1.png`（缺作品名）
  - `其他证明_铳火_a.png`（序号不是数字）

如果文件名不符合规则：
- 选择文件后会**立即在表单内提示错误**
- 不允许提交
- 提交时后端也会再次校验

### 4. 使用要求
在进入 UC 投诉前，必须确认：
- 当前投诉作品已存在于作品覆盖列表
- 代理主体(司内) 正确
- 被代理人信息正确
- 内容类型正确
- 投诉类型正确
- 作品权属文件已上传

如果作品未覆盖，系统在使用模版时可能无法匹配到对应资料。

## 六、UC 投诉操作说明

## 1. UC 投诉前的标准准备顺序
进入 UC 投诉前，建议固定按以下顺序检查：

### 第一步：检查投诉账号管理
确认：
- 已添加本次要使用的 **平台（UC）** 和 **投诉账号**
- Cookie 为最新且有效
- 如 Cookie 失效，先在 **投诉账号管理** 页面更新

### 第二步：检查被代理人信息管理
确认：
- 本次使用的 **被代理人信息已覆盖**
- 被代理人资料完整
- 授权委托书、营业执照、授权期限等信息正确

如果没有覆盖，先添加被代理人。

### 第三步：检查作品覆盖列表
确认：
- 当前投诉作品已存在
- 作品资料与本次投诉目标一致

如果没有覆盖，先新增作品。

完成以上三步后，再进入 **UC投诉** 页面发起投诉。

---

## 2. UC 投诉页面各区域功能说明

### （1）账号信息
包含：
- **采集账号**：选择本次实际发起投诉的平台账号
- **Cookie**：根据已选账号自动带出，不需要手工输入
- **本次投诉使用的被代理人信息**：选择本次实际使用的被代理人

#### 说明
- 采集账号必须先在 **投诉账号管理** 中配置
- 被代理人列表会跟随采集账号变化
- 本次投诉使用的被代理人信息是当前任务的权威值，上传自定义模版时会校验一致性

### （2）投诉模版
包含两个按钮：
- **下载自定义模版**
- **上传自定义模版**

#### 用途
用于批量载入投诉表单内容及投诉数据。

上传成功后，系统会自动：
- 填充身份信息
- 填充被代理人信息
- 填充投诉类型、功能模块、内容类型
- 填充投诉内容描述
- 加载证明文件、其他证明文件
- 生成用于投诉的 Excel 数据
- 锁定除账号信息外的主要字段，避免误改

#### 模版上传时的校验
为防止资料对不上，模版上传时会进行多重校验：

- **本次投诉使用的被代理人信息** 必须与模版中的被代理人一致
- 系统会用 **作品名称 + 被代理人信息** 联合匹配作品覆盖列表
- 命中后再结合 **代理主体(司内) + 内容类型 + 投诉类型** 进一步收窄
- 任意一条不一致都会拦截上传，并提示具体原因

> 也就是说：作品名一致还不够，被代理人不一致时也不会通过。

### （3）身份信息
展示内容包括：
- 您的身份
- 代理人/权利人
- 被代理人（权利人）信息

当前 UC 场景通常以 **代理人** 方式发起投诉。

### （4）投诉类型
当前页面主要使用：
- 投诉大类：**知识产权**
- 投诉类型：**著作权（含视频、图文、图集等）**

### （5）功能模块
根据模版或业务需要选择，如：
- 头条内容
- 大鱼号账号
- UC网盘
- 神马搜索

### （6）内容类型
根据作品实际类型选择，如：
- 影视剧集
- 其他视频
- 小说
- 漫画
- 图片
- 文章
- 软件/游戏
- 其他

### （7）投诉内容
包含：
- **批量导入 Excel**：系统按 200 行 / 批自动拆分并提交
- **投诉内容描述**：投诉原因说明

#### Excel 说明
- 支持 `.xlsx`、`.xls`
- 上传后系统会校验行数并提示预计拆分批次
- 例如 1~200 行为 1 批，201~400 行为第 2 批，以此类推

### （8）证明材料
包含：
- **证明文件**：必传
- **其他证明**：选传，可上传多个

如果通过自定义模版上传，系统会自动匹配作品相关证明、授权委托书、营业执照等资料。

## 3. UC 投诉推荐操作流程

### 方式一：使用自定义模版（推荐）
1. 进入 **UC投诉** 页面
2. 在 **账号信息** 区域选择采集账号
3. 选择 **本次投诉使用的被代理人信息**
4. 点击 **上传自定义模版**
5. 系统自动加载模版内容并锁定主要字段
6. 检查页面自动填充内容是否正确
7. 点击 **提交投诉**
8. 系统会先验证 Cookie，再创建投诉任务
9. 提交成功后，记录任务编号
10. 回到首页，在 **UC状态查询** 中查看任务进度和投诉单号

### 方式二：手工填写 + 上传 Excel
1. 进入 **UC投诉** 页面
2. 选择采集账号
3. 选择本次投诉使用的被代理人信息
4. 手工填写 / 核对投诉相关字段
5. 上传 Excel 批量导入文件
6. 上传证明文件和其他证明文件
7. 填写投诉内容描述
8. 点击 **提交投诉**
9. 查看首页状态查询结果

---

## 4. UC 投诉提交后的处理逻辑

提交投诉后，系统会：
1. 校验 Cookie 是否有效
2. 保存本次提交资料
3. 将 Excel 自动按 **200 行 / 批** 拆分
4. 按批次排队执行自动化投诉
5. 执行完成后，写入投诉单号
6. 在首页 **UC状态查询** 中展示状态和日志

### 投诉单号获取说明
当前系统已优化投诉单号获取方式：
- 不再单纯读取投诉列表最新一条
- 而是直接调用 UC 投诉接口，根据 **作品名称 + 提交时间窗口** 联合匹配本次投诉记录
- 可降低在同一账号下其他人工投诉干扰导致单号错位的问题
- 如果接口未匹配到符合数量的投诉单号，系统会标记任务失败并提示人工核对，不会再返回错误单号

---

## 5. UC 投诉注意事项

### （1）必须先配置账号
如果 **投诉账号管理** 中没有对应 UC 账号，无法正常投诉。

### （2）Cookie 失效要先更新
如果 Cookie 过期，提交时会验证失败，需先到 **投诉账号管理** 更新 Cookie。

### （3）必须核对被代理人信息
- 被代理人必须与本次投诉实际使用主体一致
- 被代理人信息需与模版内容保持一致
- 授权资料需在有效期内

### （4）必须确认作品已覆盖
如果当前投诉作品在 **作品覆盖列表** 中不存在，或对应的被代理人信息没有匹配上，模版上传会被拦截。需先到作品覆盖列表新增对应作品，并确保填入了正确的被代理人信息。

### （5）Excel 文件必须符合要求
- 仅支持 `.xlsx` / `.xls`
- 内容需符合系统要求
- 系统会自动按 200 行分批处理

### （6）证明文件不能为空
投诉提交时：
- **证明文件必传**
- 其他证明可按业务情况补充

### （7）提交成功不等于投诉立即完成
点击提交后，只代表任务已进入系统处理流程。
需到首页 **UC状态查询** 查看：
- 是否等待中
- 是否执行中
- 是否已完成
- 是否失败

### （8）失败时优先看日志
如果状态显示失败：
1. 回首页进入 **UC状态查询**
2. 点击 **查看日志**
3. 根据日志排查 Cookie、资料、模版、网络或平台页面变化等问题

### （9）单 Worker 串行执行
当前系统 UC 投诉任务是排队串行执行的：
- 一次只跑一个任务
- 多个任务提交后会依次处理
- 首页中“预估完成时间”可作为参考

## 七、推荐标准操作顺序（简版）

每次做 UC 投诉时，建议固定按下面顺序操作：

1. **首页 → 投诉账号管理**
   - 检查是否已添加 UC 账号
   - 检查 Cookie 是否最新

2. **首页 → 被代理人信息管理**
   - 检查本次需要使用的被代理人是否已覆盖
   - 核对授权资料和营业执照是否完整

3. **首页 → 作品覆盖列表**
   - 检查当前投诉作品是否已覆盖
   - 如未覆盖，新增作品

4. **首页 → UC投诉**
   - 选择采集账号
   - 选择本次投诉使用的被代理人信息
   - 上传自定义模版或手工填写
   - 提交投诉

5. **首页 → UC状态查询**
   - 刷新查看执行状态
   - 查看投诉单号
   - 必要时查看日志

## 八、常见问题

### 1. 选择不到投诉账号
说明该平台账号尚未在 **投诉账号管理** 中配置，需要先添加。

### 2. 选择不到被代理人信息
说明该账号下尚未配置对应被代理人，需要先到 **被代理人信息管理** 中添加。

### 3. 模版上传失败
常见原因：
- 未先选择采集账号
- 模版内容格式不符合要求
- 模版中的被代理人信息与当前选择不一致
- 模版对应作品未在作品覆盖列表中建立，或建立时填写的被代理人信息和模版不一致
- 上传的证明文件 / 营业执照 / 授权委托书 命名不符合规则

### 4. 提交时提示 Cookie 验证失败
说明当前账号 Cookie 已过期或失效，需要去 **投诉账号管理** 页面更新。

### 5. 提交成功但首页还没显示投诉单号
可能原因：
- 任务还在排队或执行中
- 批次尚未全部完成
- 平台接口入库存在短暂延迟

可稍后刷新首页 **UC状态查询** 查看。
