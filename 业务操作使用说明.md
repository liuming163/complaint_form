# 业务操作使用说明

## 一、系统用途

本系统用于投诉业务的资料管理与自动化提交，当前支持：
- 投诉账号管理
- 被代理人信息管理
- 作品覆盖列表
- UC 投诉（支持多作品批量投诉）
- 百度投诉（百度网盘/搜索版权批量投诉）
- 首页状态查询（UC / 百度）

建议每次正式投诉前，先按顺序完成以下检查：
1. 检查投诉账号是否已配置、Cookie 是否有效
2. 检查被代理人信息是否已覆盖且资料完整（UC 投诉需要）
3. 检查当前投诉作品是否已覆盖（UC 投诉需要）
4. 再进入对应投诉页面发起投诉

---

## 二、首页功能说明

进入系统首页后，可看到两个主要区域：

### 1. 顶部导航区
顶部可直接进入以下页面：
- **投诉账号管理**：维护投诉平台、投诉账号、Cookie、账号用途
- **被代理人信息管理**：维护被代理人名称、营业执照、授权委托书等资料
- **作品覆盖列表**：维护作品名称、代理主体(司内)、被代理人信息、内容类型、投诉类型、作品权属文件等

### 2. 左侧「投诉平台」区域
用于进入实际投诉页面：
- **UC投诉**：UC 平台投诉表单（支持多作品批量投诉）
- **百度投诉**：百度网盘/搜索版权批量投诉
- **夸克投诉**：暂未开通

### 3. 右侧「状态查询」区域
用于查看任务执行状态：
- **UC状态查询**：可查看 UC 投诉任务的提交时间、采集账号、作品名称、投诉单号、脚本执行状态、日志入口
- **百度状态查询**：可查看百度投诉任务的提交时间、采集账号、作品名称、投诉单号、执行状态、日志入口

#### 状态查询通用说明
- 点击 **刷新** 可拉取最新状态
- 任务状态包括：**等待中 / 执行中 / 已完成 / 失败**
- 若日志仍可用，可点击 **查看日志** 排查问题
- 如果显示 **日志已过期**，说明日志保留期已过
- 支持 **导出Excel** 查看详细投诉结果

## 三、投诉账号管理

### 1. 页面用途
用于维护实际发起投诉所使用的平台账号信息，包括：
- 平台名称
- 代理主体(司内)
- 投诉账号
- 账号用途
- Cookie

### 2. 页面字段说明
账号列表中展示：
- **平台名称**：当前支持 UC、百度
- **代理主体(司内)**：如和晞科技
- **投诉账号**：实际登录投诉平台的账号
- **账号用途**：便于区分账号使用场景
- **Cookie**：投诉时自动带入，用于身份验证。列表中只展示开头与结尾的若干位、中间用省略号；鼠标悬停可看完整内容

### 3. 操作说明
#### （1）新增投诉账号
1. 点击 **添加账号**
2. 选择 **代理主体(司内)**
3. 选择 **平台名称**（UC / 百度）
4. 填写 **投诉账号**
5. 粘贴完整 **Cookie**
6. 选填 **账号用途**
7. 点击 **确认添加**

#### （2）更新 Cookie
1. 在列表中找到对应账号
2. 点击 **更新Cookie**
3. 粘贴新的完整 Cookie
4. 点击 **确认更新**

### 4. 使用要求
进行投诉前，**必须先检查本页面**：
- 是否已经添加了需要使用的平台和投诉账号

## 四、被代理人信息管理

### 1. 页面用途
用于维护投诉时所使用的被代理人（权利人）信息，以及相关证明资料。

> 注意：百度投诉无需添加被代理人，添加被代理人弹窗中百度选项已禁用。

### 2. 页面展示内容
页面列表按账号维度展示：
- 平台名称
- 代理主体(司内)
- 投诉账号
- 被代理人信息
- **被代理人营业执照**：列表中显示文件名，点击可在新窗口预览
- **授权委托书**：列表中显示文件名，点击可在新窗口预览

### 3. 添加被代理人流程
进行 UC 投诉前，需要检查当前要使用的被代理人信息是否已覆盖。

如果未覆盖：
1. 点击 **添加被代理人**
2. 选择 **代理主体(司内)**
3. 选择 **平台名称**（百度选项已禁用，无需添加）
4. 选择 **投诉账号**
5. 输入 **被代理人信息**
6. 按需上传资料：
   - 被代理人营业执照
   - 授权委托书
   - 授权期限截止日期
7. 点击 **确认添加**

#### 上传文件命名规则（重要）
上传时会**对文件原始名称做即时校验**：

- **被代理人营业执照** 必须命名为：
  ```
  营业执照_被代理人信息.xxx
  ```
  例如：`营业执照_深圳阅荣信息科技有限公司.png`

- **授权委托书** 必须命名为：
  ```
  授权委托书_被代理人信息_代理主体(司内)_截止日期YYYYMMDD.xxx
  ```
  例如：`授权委托书_深圳阅荣信息科技有限公司_和晞科技_截止日期20260605.pdf`

如果文件名不符合规则，选择文件后会立即提示错误，不允许提交。

### 4. 页面逻辑说明
- 被代理人资料是按 **平台 + 代理主体(司内) + 投诉账号 + 被代理人名称** 进行管理的
- 已存在的资料在本页面中只展示，不允许直接覆盖上传
- 缺失部分可以继续补充

## 五、作品覆盖列表

### 1. 页面用途
用于维护当前可用于 UC 投诉的作品资料。

每条作品资料至少关联：
- 剧名
- 代理主体(司内)
- 被代理人信息
- 内容类型
- 投诉类型
- 作品权属文件

### 2. 列表说明
页面列表可查看：
- **剧名**
- **代理主体(司内)**
- **被代理人信息**
- **内容类型**
- **投诉类型**
- **作品权属文件**：文件名可点击，新窗口预览
- **其他证明**：当存在其他证明文件时，会显示 `查看(N)` 按钮

支持搜索：剧名、代理主体(司内)、被代理人信息、内容类型、投诉类型

### 3. 新增作品流程
1. 点击 **新增作品**
2. 填写 **剧名**
3. 选择 **代理主体(司内)**
4. 选择 **被代理人信息**
5. 选择 **内容类型**
6. 选择 **投诉类型**
7. 上传 **作品权属文件**（必传）
8. 按需上传 **其他证明文件**（最多 2 个）
9. 点击 **提交**

#### 上传文件命名规则（重要）
- **作品权属文件** 必须以下列前缀开头：
  ```
  证明文件_作品名称
  ```
  例如：`证明文件_铳火.png`、`证明文件_铳火_file.jpg`

- **其他证明文件** 必须严格符合：
  ```
  其他证明_作品名称_序号.xxx
  ```
  例如：`其他证明_铳火_1.png`、`其他证明_铳火_2.jpg`

### 4. 使用要求
在进入 UC 投诉前，必须确认当前投诉作品已存在于作品覆盖列表，且代理主体、被代理人信息、内容类型、投诉类型、作品权属文件均正确。

## 六、UC 投诉操作说明

### 1. UC 投诉特点
- 支持 **多作品批量投诉**：一次提交可包含多部作品
- 系统自动按 **200 行/批** 拆分 Excel 数据
- 每部作品独立匹配证明文件
- 未匹配到的作品会提示跳过，不影响其他作品投诉

### 2. UC 投诉前的标准准备顺序

#### 第一步：检查投诉账号管理
- 已添加 UC 平台投诉账号

#### 第二步：检查被代理人信息管理
- 本次使用的被代理人信息已覆盖
- 授权委托书、营业执照、授权期限等信息正确

#### 第三步：检查作品覆盖列表
- 当前投诉作品已存在
- 作品资料与本次投诉目标一致

### 3. UC 投诉页面各区域功能说明

#### （1）账号信息
- **采集账号**：选择本次实际发起投诉的平台账号
- **Cookie**：根据已选账号自动带出
- **本次投诉使用的被代理人信息**：选择本次实际使用的被代理人

#### （2）投诉模版
- **下载自定义模版**：下载 Excel 模版用于填写投诉数据
- **上传自定义模版**：上传填好的模版，系统自动解析并加载

上传成功后，系统会自动：
- 填充身份信息、被代理人信息
- 填充投诉类型、功能模块、内容类型
- 填充投诉内容描述
- 加载证明文件、其他证明文件
- 按作品拆分生成批次数据
- 锁定除账号信息外的主要字段

#### 模版上传时的校验
- **本次投诉使用的被代理人信息** 必须与模版中的被代理人一致
- 系统会用 **作品名称 + 被代理人信息** 联合匹配作品覆盖列表
- 命中后再结合 **代理主体(司内) + 内容类型 + 投诉类型** 进一步收窄
- 不一致会拦截上传并提示具体原因

#### （3）身份信息
- 您的身份
- 代理人/权利人
- 被代理人（权利人）信息

#### （4）投诉类型
- 投诉大类：知识产权
- 投诉类型：著作权（含视频、图文、图集等）

#### （5）功能模块
如：头条内容、大鱼号账号、UC网盘、神马搜索

#### （6）内容类型
如：影视剧集、漫画、小说、其他

#### （7）投诉内容
- **批量导入 Excel**：系统按 200 行/批自动拆分并提交
- **投诉内容描述**：投诉原因说明

#### （8）证明材料
- **证明文件**：必传
- **其他证明**：选传，可上传多个

通过自定义模版上传时，系统会自动匹配作品相关证明、授权委托书、营业执照等资料。

### 4. UC 投诉推荐操作流程

1. 进入 **UC投诉** 页面
2. 选择采集账号
3. 选择 **本次投诉使用的被代理人信息**
4. 点击 **上传自定义模版**
5. 系统自动加载模版内容并锁定主要字段
6. 检查页面自动填充内容是否正确
7. 点击 **提交投诉**
8. 系统验证 Cookie 后创建投诉任务
9. 提交成功后显示任务编号、投诉作品数、总链接数、总批次数
10. 回到首页，在 **UC状态查询** 中查看任务进度

### 5. UC 投诉提交后的处理逻辑

提交投诉后，系统会：
1. 校验 Cookie 是否有效
2. 保存本次提交资料
3. 将 Excel 按作品拆分，每作品再按 **200 行/批** 拆分
4. 按批次排队执行自动化投诉
5. 每个批次完成后自动切换到下一部作品（替换证明文件）
6. 执行完成后，写入投诉单号
7. 在首页 **UC状态查询** 中展示状态和日志

### 6. UC 投诉注意事项

- Cookie 失效要先更新，否则提交时验证失败
- 被代理人信息必须与模版内容保持一致
- 未在作品覆盖列表中匹配到的作品会被跳过，不影响其他作品
- 跳过的作品信息可在首页 UC 状态查询中导出查看
- 当前系统 UC 投诉任务是排队串行执行的，一次只跑一个任务

## 七、百度投诉操作说明

### 1. 百度投诉特点
- 无需配置被代理人信息（百度平台不需要）
- 无需在作品覆盖列表中添加作品
- 提交前会自动 **预检作品权属状态**（调用百度接口查询是否有已通过审核的权属记录）
- 未通过权属审核的作品会提示跳过

### 2. 百度投诉前的准备
只需确认：
- 已在 **投诉账号管理** 中添加百度平台账号

### 3. 百度投诉页面功能说明

#### （1）账号信息
- **采集账号**：选择百度平台投诉账号
- **Cookie**：根据已选账号自动带出
- **验证Cookie**：点击后验证 Cookie 是否有效，验证通过后才能上传模版

#### （2）投诉模版
- **下载自定义模版**：下载百度投诉专用 Excel 模版
- **上传自定义模版**：上传填好的模版，系统解析后显示作品预览

#### （3）作品预览
模版上传成功后显示：
- 投诉产品（如百度网盘）
- 作品数量
- 总链接数
- 总批次数
- 每部作品的名称、投诉描述、原版链接、侵权链接数、批次数

#### （4）提交投诉
点击提交后，系统会：
1. **预检作品权属状态**：调用百度接口查询每部作品是否有已通过审核的权属记录
2. 如有未通过的作品，提示用户确认是否跳过
3. 提交可投诉的作品，创建投诉任务

### 4. 百度投诉推荐操作流程

1. 进入 **百度投诉** 页面
2. 选择采集账号
3. 点击 **验证Cookie**，确认验证通过
4. 点击 **下载自定义模版**，填写投诉数据
5. 点击 **上传自定义模版**
6. 检查作品预览信息是否正确
7. 点击 **提交投诉**
8. 系统自动预检权属状态，确认后提交
9. 回到首页，在 **百度状态查询** 中查看任务进度

### 5. 百度投诉注意事项

- 百度投诉不需要添加被代理人信息
- 百度投诉不需要在作品覆盖列表中添加作品
- 提交前系统会自动检查作品权属状态，未通过审核的作品无法自动投诉
- 未通过权属审核的作品需在百度投诉原平台手动进行投诉
- Cookie 验证通过后才能上传模版

## 八、推荐标准操作顺序（简版）

### UC 投诉

1. **首页 → 投诉账号管理**
   - 检查是否已添加 UC 账号

2. **首页 → 被代理人信息管理**
   - 检查本次需要使用的被代理人是否已覆盖
   - 核对授权资料和营业执照是否完整

3. **首页 → 作品覆盖列表**
   - 检查当前投诉作品是否已覆盖
   - 如未覆盖，新增作品

4. **首页 → UC投诉**
   - 选择采集账号和被代理人信息
   - 上传自定义模版
   - 提交投诉

5. **首页 → UC状态查询**
   - 刷新查看执行状态和投诉单号

### 百度投诉

1. **首页 → 投诉账号管理**
   - 检查是否已添加百度账号

2. **首页 → 百度投诉**
   - 选择采集账号
   - 验证 Cookie
   - 上传自定义模版
   - 提交投诉

3. **首页 → 百度状态查询**
   - 刷新查看执行状态和投诉单号

## 九、常见问题

### 1. 选择不到投诉账号
说明该平台账号尚未在 **投诉账号管理** 中配置，需要先添加。

### 2. 选择不到被代理人信息（UC）
说明该账号下尚未配置对应被代理人，需要先到 **被代理人信息管理** 中添加。

### 3. UC 模版上传失败
常见原因：
- 未先选择采集账号
- 模版内容格式不符合要求
- 模版中的被代理人信息与当前选择不一致
- 模版对应作品未在作品覆盖列表中建立
- 上传的证明文件命名不符合规则

### 4. 提交时提示 Cookie 验证失败
说明当前账号 Cookie 已过期或失效，需要去 **投诉账号管理** 页面更新。

### 5. 百度投诉提示"未找到已通过审核的权属记录"
说明该作品在百度平台上没有已通过审核的权属认证，需要先在百度投诉原平台手动完成权属认证后再使用本系统投诉。

### 6. 提交成功但首页还没显示投诉单号
可能原因：
- 任务还在排队或执行中
- 批次尚未全部完成
- 平台接口存在短暂延迟

可稍后刷新首页状态查询查看。

### 7. UC 投诉部分作品被跳过
说明这些作品在作品覆盖列表中未匹配到或证明文件不齐全。可在首页 UC 状态查询中导出 Excel 查看具体原因，补充资料后重新投诉。
