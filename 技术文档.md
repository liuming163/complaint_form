## 二、项目简介

- **项目名称**：投诉处理系统
- **目标**：在 Web 端集中管理投诉账号、被代理人资料、作品资料；接收投诉表单后通过浏览器自动化（UC）或 API 调用（百度、夸克）批量提交到对应投诉平台。
- **运行形态**：单体 Flask 服务 + 单 Worker 串行执行任务（统一队列消费 UC、百度、夸克任务）。

## 三、技术栈

| 层 | 选型 |
|---|---|
| Web 框架 | Flask |
| 数据访问 | SQLAlchemy 2.x（使用 `text(...)` 直接执行原生 SQL） |

| 配置管理 | python-dotenv 加载项目根目录 `.env` |
| 前端 | 原生 HTML + Bootstrap 5 + Fuse.js（前端模糊搜索） |

## 七、运行所需的组件

正常运行需要 3 个组件：

1. **MySQL**：业务数据库
2. **Redis**：投诉任务队列与 Worker 锁
3. **Playwright + Chromium**：UC 自动化浏览器（百度投诉不需要）

外加 2 个进程：

- `app.py`：Web 服务，监听 5001
- `worker.py`：消费 Redis 统一队列，根据任务类型分发到 UC（Playwright）、百度（API）或夸克（API）

---

## 八、本地启动方式

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 安装 Playwright 浏览器
playwright install chromium

# 3. 启动 Redis（任选其一）
brew services start redis           # macOS 推荐
redis-server                        # 前台启动

# 4. 启动 Web
python3 app.py                      # 监听 http://127.0.0.1:5001

# 5. 启动 Worker（另开一个终端）
python3 worker.py
```

> Worker 同一时间只能存在一个，第二个进程会因抢不到 Redis 锁直接退出。

## 十、投诉执行流程

### UC 投诉（Playwright 浏览器自动化）
1. 用户提交 → `app.py` 落库 + 入 Redis 队列
2. `worker.py` 消费任务 → subprocess 调用 `uc_complaint_from_backend.py`
3. 脚本启动 Chromium → 登录 UC 投诉平台 → 填写表单 → 上传证明文件 → 批量导入 Excel → 提交
4. 多作品场景：每部作品完成后替换证明文件，继续下一部
5. 结果写入 `task_results/` + 更新数据库状态

### 百度投诉（API 调用）
1. 用户提交 → `app.py` 落库 + 入 Redis 队列
2. `worker.py` 消费任务 → subprocess 调用 `baidu_complaint_backend.py`
3. 脚本通过 requests 调用百度版权保护 API → 上传侵权链接 → 提交投诉
4. 结果写入 `task_results/` + 更新数据库状态

> 百度支持多投诉产品（百度网盘 / 百度搜索 / 百度APP / 百家号 / 百度贴吧 / 好看视频等），投诉产品到 `complaint_type` 编码的映射见 `app.py` 的 `BAIDU_COMPLAINT_TYPE_MAP`；投诉产品为「好看视频」时需额外提供「侵权类型」（`BAIDU_INFRINGE_TYPE_MAP`）。

### 夸克投诉（API 调用）
1. 用户提交 → `quark_routes.py` 的 `/api/quark/submit` 落库 + 入 Redis 统一队列（`platform='quark'`）
2. `worker.py` 消费任务 → subprocess 调用 `quark_complaint_backend.py`
3. 脚本通过 requests 调用夸克版权投诉 API（`https://ipp.quark.cn`）→ 按作品分组、按 200 行/批提交侵权链接
4. 结果写入 `task_results/`（`quark_*.log`）+ 更新数据库状态（含每作品 `feedback_numbers`）

> 夸克功能模块、内容类型、投诉大类/类型到编码的映射见 `quark_routes.py` 中的 `QUARK_MODULE_NAME_MAP` / `QUARK_CONTENT_TYPE_NAME_MAP` / `QUARK_TYPE_MAP` / `QUARK_SUB_TYPE_MAP`。模板上传时会按「作品名 + 被代理人 + 代理主体」匹配 `works` 表，并在 `static/imgs/剧名/` 下匹配证明文件、授权委托书、营业执照。

## 十二、常见运行问题

| 现象 | 优先排查 |
|---|---|
| 页面打不开 | 看 `app.py` 终端日志 / 5001 端口是否被占 |
| 提交后任务不跑 | Redis 是否启动；`worker.py` 是否在跑；锁 key 是否残留 |
| Worker 启动报 `Another Redis worker is already running` | 已有 worker 在跑或锁未释放，等 TTL 过期或 `redis-cli del uc_complaint_worker_lock` |
| UC 任务失败 | 在首页 UC 状态查询点"查看日志"，定位是 Cookie / 资料 / 平台变化哪类 |
| 百度任务失败 | 在首页百度状态查询点"查看日志"，常见原因：Cookie 过期、权属未通过、API 限流 |
| 夸克任务失败 | 在首页夸克状态查询点"查看日志"（`task_results/quark_*.log`），常见原因：Cookie 过期、作品未覆盖/缺证明文件、API 限流 |
| 投诉单号没出来 | 检查作品名称、提交时间窗内平台是否真的入库 |
| 首页状态查询报错 | 检查数据库连接是否正常 |
